    3.ETIQUETAS Y ATRIBUTOS

    Las etiquetas HTML van a ayudarnos a brindar una estructura y semántica al contenido de nuestro website y cada una tiene características y usos diferentes aunque visualmente den un resultado similar. Los atributos le dan características extra a las etiquetas para complementar información y son pre programadas por el navegador, algunos atributos funcionan en todas las etiquetas de la misma forma como ‘class’ pero otros atributos funcionan en solo alguna etiquetas como ‘src’

    a. Etiquetas

    La estructura de las etiquetas siempre tiene una apretura <(nombre de la etiqueta)> luego el contenido y por ultimo el cierre < / nombre de la etiqueta >

    No todas las etiquetas necesitan abrirse y cerrarse 8por ejemplo img, se cierra en ella misma, no tiene ningun contenido, pero aún así el navegador la entiende. )

    - h1: Representa el mensaje más importante del website (nombre base). Hay mas etiquetas de h, Heading o cabecera, van desde la h1 hasta la h6.

    - p: Con ella separaremos los párrafos

    - a: inserta un enlace. Debemos especificar el nombre (contenido). Si queremos que al hacer click en el enlace se abra una pestaña nueva, lo haremos a través del atributo TARGET. tambien puedo crear en mis menus de navegación ENLACES INTERNOS. Primero creamos nuestra etiqueta de ancla (a) y para vincular a un elemento dentro de la pagina tengo que poner # y luego tengo que hacerle referencia a un elemento con id (cualquier etiqueta puede tener un id, es un atributo, pero cada id tiene que ser unico por pagina ej: google chromr, solo puede existir uno por pagina, tendriamos que cambiar el nombre al segundo)

    También podemos insertar un mail que vinvule un correo electronico para que nos escriban drectamente. Siempre entre comillas ej: a.href="mailto:correoelectronico"

    - hr: representa un cambio de tema entre párrafos. Visualmente es una linea

    - div: Sirve para crear secciones o agrupar contenidos. No se debe usar sin sentido, es simplemente un grupo de elementos, pero no aporta información desde el punto de vista de la aacesibilidad o los motores de busqueda.

    - nav: Es una etiqueta semántica que le dice a mi navegador que una lista es un menú de navegación. Visualmente no hay nada, pero en cuanto a accesibilidad da mas información.

    - header:los headers suelen ir en la parte superior del diseño, pero no es necesarioq ue siempre sea así. Es la navegación pincipal del tu contenido, la cabecera.

    - details y summary: Son etiquetas que siempre deben ir una dentro de la otra. summary nos da el nombre que acompaña a la flecha el menu desplegable (va siempre dentro de details) y dentro de details pondremos la lista. open es un atributo para agregar a details, para que el menu este abuerto por defecto

    - Existen dos etiquetas para poner codigo:

    code: Se suele utilizar para introducir una porcion de codigo pequeña dentro de un parrafo (dentro de la etiueta parrafo), hace una señalizacion en linea. Es una etiqueta con función más semántica que visual.

    pre: Se utiliza para escribir codigo en bloque que ocupe toda el area (todo el ancho o espacio) disponible. Es conveniente cuando queremos escribir una porcion grande de codigo. Lo primero que hace pre es tomar todos los espacios que tenemos hasta la indentación. si queremos que el codigo este totalmente alibneado a la izquierda, quitariamos esos espacios. Esta etiqueta esta hhecha para que pongamos pedazos de codigo, para hacer esto existen las entidades html donde puedo representar ciertos caracteres, para que representen otras cosas, pero en texto. Los guiente es hacer un proceso llamado "escapar del codigo". Para hacerlo mas rapido podemos utilizar la pagina freeformatter

    4. SEMÁNTICA

    La semántica le brinda sentido a cada elemento existente en un sitio web, algunos elementos será meramente decorativos y no deben significar nada pero otros serán títulos y deberán estar en diferente jerarquías o alguna secciones serán más relevantes que otras, esto ayudará a los motores de búsqueda como Google o Duck Duck Go a diferenciar tu contenido, categorizar y será la herramientas más valiosa para estar en los primeros resultados de búsqueda SEO. También ayudará a la accesibilidad de tu sitio web, para que personas con habilidades diferentes puedan entender cada contenido. Una de las cosas mas importantes en la accesibilidad es tener una BUENA SEMÁNTICA.

    5. QUE ES CSS?

    CSS Cascading Style Sheets es el lenguaje de estilos utilizado para describir la presentacion de documentos HTML. Podemos incluir CSS u hojas de estilo en HTML de formas:

    - Como atributo: style

    - Como etiqueta: style. Puede ir en head o en cualquier parte.

    - Mediante archivo externo: mediante la etiqueta "link" podemos enlazar un archivo de hojas de estilo externo. Lo hacemos para no mezclar CSS y html y no hacer mas dificil la busqueda dentro de html.

    dentro de VSC creamos una nueva hoja (ctrl+n) y la guardamos (se puede usar cualquien nombre, en mi caso styles.css) pero es importante que la extención del nombre siempre sea .css

    Reglas de CSS:

    Una regla es la forma en la que se estructura para poner un tipo de estilo. Se compone de un selector (o varios separados por ,), una propiedad (o varias) y un valor.

    SELECTORES

    Los selectores definen sobre que conjunto de elementos se aplicará un conjunto de reglas CSS

    Basicos:

    - *global: selectores universales o globales (l). Afecta a todos los elementos

    - De Etiquetas: Hacen referencia a todoslos elementos HTML qur coincidan. Se escrbe igual que una etiqueta sin el <>. Si lo utilizamos como atributo, solo le va a dar estilo a ese tag o elemento

    - .clase: Las clases se pueden repetir y un elemento puede tener mas de una clase. puedes poner cuantas clases quieras a un elemnento HTML.

    - # id: Solo un elemento puede tener el mismo id en una pagina. Un elemento de id se identifica con #

    - [atributo]: Todos los eleentos que contengan el atributo o valor de atributo en cuestión. Se utiliza cuando ya no existe un forma mas sencilla de hacer un selector

    - de grupo: N selectores se pueden configurar con las mismas propiedades. Podemos darle estilo a mas de un elemento con la misma regla.

    Combinados: Con estos selectores podemos ser mas especificos con respecto a los elementos con los que estamos buscando para darles algún tipo de estilo diferente. son descendentes.

    - descendente: más específico.

    - >Hijo directo: Los hijos directos del selector previo a > . Le voy a dar estilo solamanete al primer que yo defina y que cumpla con determinadas condiciones.

    - ~hermano: Es para los elemtos en el mismo nivel. Es decir para el elemento que esta al costado del otro. ej: h1~h2{color: red} todos los h2 hermanos del h1.

    - +adyacente: Como el hermano, pero solo el mas cercano.

    Pseudoselectores: Son estados que solo pueden pasarle a un elemento.

    - :Pseudoclase:Representan estados en un elemnto. Ej: cambiar de color un link ya visitado o pcambiar de color al pasar el mouse encima de un link.

    - ::pseudoelemento: Crean elemntos especiales que no se pueden crear con HTML, agraga algo de contenido.

    CASCADA, ESPECIFICIDAD y HERENCIA

    Cascada: Como su nombre indica CSS Cascade style sheet, CSS seleede arriba hacia abajo y lo que venga despues sobreescribe lo anterior, si una segunda regla con el mismo selector declara la misma propiedad, esta segunda sobreescribira la primera.

    Especificidad:Cuando hay coincidencia de elementos se aplica la cascada y tambien la relevancia que tenga ese selector. ej: Class es mayor que Tag. Existe una propiedd que se puede añadir llamada "important" te sobreescribe cualquier cosa, pero no debe ser usado a la ligera, solo debe usarse si es estrictamente necesario.

    Herencia: Existen propiedades de CSS que se heredan

    CONTENT-COMPONENT-LAYOUT

    - Contenido: cada uno de los elementos de la pagina (los detalles, ej: una imagen, un boton, un texto)

    - Componente: Es un conjunto de elementos, ej: un menu, una lista.

    - Layout: es la interfaz. un conjunto de componentes

    DISPLAY /FLUJO /LINEA Y BLOQUE

    - Display: la propiedad CSS display (display:) especifica si un elemento es tratado como block (espera ocupar todo el espacio de su linea) o inline elelement (ej: img, solo ocupa s espacio fisica en la linea. Si pongo una imagen y enseguida en la linea otra, se pondran una al lado de la otra.) y el diseño usado por sus hijos, como flor layut (diseño de flujo ) flex (flexible) o grid (cuadricula).

    - Flow: Es el flujo ntural de nuestro navegador. obedece a la configuración del witing-mode y direction, ambas sn propiedades de CSS con un valor por defect. El writing mode es Horizontal-tb: Se lee en horizontal (algunos idiomes se escriben y leen en vertical) y "t"-"b" significa top-bottom de arriba hacia abajo. Direction: ltr, ltr significa left to right (de derecha a izquierda, el arabe por ejemplo, seria de izq a der.)

    Ecisten valores o unidades relativas o absolutas. Hay valores absolutos y relativos para interpretar valores (por ejemplo el tamaño de letra se puede representar en EM), los pixels son unidades absolutas.

    - em: Es una unidad relativa. Es igual al valor de fuente mas cercano del selector que estas ocupando. Va en herencia, toma el valor del padre mas cercano.

    - rem: Es una unidad relativa. Siempre va a tomar el valor o tamaño del elemento que este mas cerca al RUT de nuestro documento.
